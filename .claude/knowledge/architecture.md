# スポーツタイマー 実装仕様書 (architecture.md)

本書は `.claude/CLAUDE.md` に記載された概要と、`スポーツタイマー画面レイアウト.drawio` の画面レイアウト、
およびユーザーへのヒアリング結果をもとに、実装可能なレベルまで詳細化した仕様書である。

## 1. 概要

- ブラウザで動作するシンプルなスポーツタイマー(ストップウォッチ)アプリ。
- 専用のSTART/STOPボタンは持たず、画面(ウィンドウ)のどこをクリック/タップしてもSTARTとSTOPが切り替わる。
- 記録(ラップ・履歴)機能は持たない。
- `localStorage` 等の永続化処理は一切行わない(ページをリロードすると状態は初期化される)。

## 2. 画面状態と表示

アプリの状態は以下の2状態のみを持つ。ボタンを持たないため、状態は背景色の変化で示す。

| 状態名 | 説明 | 背景色による状態表示 |
|---|---|---|
| 停止中 (STOPPED) | カウントアップしておらず、時間表示は固定 | 「停止中」を表す背景色 |
| 計測中 (RUNNING) | 1秒ごとにカウントアップ中 | 「計測中」を表す背景色 |

- ページ読み込み直後の初期状態は「停止中」、時間表示は `00:00:00`。
- 具体的な配色(何色を停止中/計測中に割り当てるか)は指定なし。実装時に、両状態が明確に見分けられる配色を決定する。

### 2.1 時間表示フォーマット

- 表示形式: `HH:MM:SS` (例: `00:00:00`)
- 各項目は常に2桁のゼロ埋め表示 (例: 5秒 → `00:00:05`)。
- 表示可能範囲: `00:00:00` 〜 `99:59:59`
- 1秒未満(ミリ秒)の表示は行わない。

### 2.2 レイアウト構成

drawio上の2つの枠(時間表示+ボタン)は当初のモックアップであり、以下の変更により
カード型のレイアウトからウィンドウ全体を使ったレイアウトに変更する。

- カード(枠付きコンテナ)は廃止する。時間表示はウィンドウ(画面)いっぱいに、背景全面を使って表示する。
- 時間表示はウィンドウの中央に、画面サイズに応じてできるだけ大きく表示する(ウィンドウ幅・高さの変化に追従してフォントサイズが変化するレスポンシブ対応とする)。
- 専用のボタンは表示しない。時間表示を含む画面全体がクリック/タップ可能な操作領域となる。
- 配色・フォント種別など見た目の詳細指定はなし(上記の状態表示用の背景色を除く)。実装時に見やすさを優先して適宜決定してよい。

## 3. 動作仕様

### 3.1 画面クリック/タップ時の挙動

- 専用のボタンは持たず、画面(ウィンドウ)内のどこをクリック/タップしても、押すたびに「開始」「停止」の動作を交互に切り替える(トグル)。

**「停止中」の状態で画面をクリック/タップした場合(開始操作):**

1. 時間表示を `00:00:00` にリセットする。
2. カウントアップを開始する(1秒ごとに+1秒)。
3. 背景色を「計測中」を表す色に変更する。
4. 状態を「計測中」にする。

**「計測中」の状態で画面をクリック/タップした場合(停止操作):**

1. カウントアップを停止する。
2. その時点の時間表示をそのまま固定表示する(次に画面がクリック/タップされるまで値は変わらない)。
3. 背景色を「停止中」を表す色に変更する。
4. 状態を「停止中」にする。

> 備考: 停止させた後に再度画面をクリック/タップした場合、一時停止からの再開(継続カウント)ではなく、
> 常に `00:00:00` からの新規カウントとなる。

### 3.2 上限到達時 (99:59:59) の挙動

- カウントアップ中に `99:59:59` に到達した場合、自動的に「停止中」状態に遷移する。
  - カウントアップを停止する。
  - 表示は `99:59:59` のまま固定される。
  - 背景色を「停止中」を表す色に変更する。
  - この後、再度画面をクリック/タップした場合の挙動は 3.1 の「停止中→クリック/タップ」と同じ(`00:00:00` から再スタート)。

### 3.3 リセット機能

- 明示的な「リセットボタン」は実装しない(操作は画面全体のクリック/タップのみ)。
- `00:00:00` に戻す手段は次のいずれか:
  - ページのリロード
  - 「停止中」の状態から画面をクリック/タップする(新規カウント開始時に自動的に0からになる)

## 4. 非機能要件

- 永続化処理(`localStorage`, `sessionStorage`, Cookie等)は一切使用しない。
- 対応デバイス: PC・スマートフォンの両方を想定したレスポンシブ対応を行う(画面幅・高さに応じて時間表示のフォントサイズ等が追従し、レイアウトが破綻しないこと)。
- クリック/タップの判定領域はウィンドウ全体(画面いっぱいに表示される時間表示を含む)とする。
- タイマーの計測方法は、`setInterval` の呼び出し回数のみに依存せず、開始時刻と現在時刻の差分計算(例: `Date.now()` の差分)により実時間との誤差が蓄積しないようにする。

## 5. 技術構成

- 構成: ブラウザで完結する静的Webアプリ(サーバーサイド処理なし)。
- ファイル構成: 役割ごとにファイルを分割する。
  - `index.html` — 画面構造(HTML)
  - `style.css` — 独自スタイル(Tailwindで表現しきれない部分がある場合のみ使用)
  - `script.js` — タイマーのロジック・状態管理・DOM操作
- Tailwind CSS: CDN経由で読み込む(ビルドツール・npmは使用しない)。
  ```html
  <script src="https://cdn.tailwindcss.com"></script>
  ```
- JavaScript: フレームワークを使用しない Vanilla JS で実装する。

```
                 [ページ読み込み]
                        │
                        ▼
        ┌───────────────────────────┐
        │   停止中 (STOPPED)         │◀───────────────┐
        │   表示: 固定値              │                 │
        │   背景色: 停止中の色         │                 │
        └───────────────────────────┘                 │
                        │ 画面クリック/タップ             │
                        │ (表示を00:00:00にリセット)      │ 画面クリック/タップ or
                        ▼                              │ 99:59:59到達(自動)
        ┌───────────────────────────┐                 │
        │   計測中 (RUNNING)          │─────────────────┘
        │   表示: 1秒ごとに+1          │
        │   背景色: 計測中の色         │
        └───────────────────────────┘
```

## 7. 未確定/今後確認が必要な事項

- 現時点で機能仕様上の未確定事項はなし。
- 「停止中」「計測中」を表す具体的な背景色の値、および数字自体のフォント種別は指定なし。実装時に、両状態が明確に見分けられ、かつ数字が読みやすい配色を決定する。
