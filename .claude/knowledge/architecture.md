# スポーツタイマー 実装仕様書 (architecture.md)

本書は `.claude/CLAUDE.md` に記載された概要と、`スポーツタイマー画面レイアウト.drawio` の画面レイアウト、
およびユーザーへのヒアリング結果をもとに、実装可能なレベルまで詳細化した仕様書である。

## 1. 概要

- ブラウザで動作するシンプルなスポーツタイマー(ストップウォッチ)アプリ。
- 画面には「時間表示」と「STARTボタン/STOPボタン」のみを持つ、単一の操作ボタン(1つのボタンがSTART/STOPを兼ねる)。
- 記録(ラップ・履歴)機能は持たない。
- `localStorage` 等の永続化処理は一切行わない(ページをリロードすると状態は初期化される)。

## 2. 画面状態と表示

アプリの状態は以下の2状態のみを持つ。

| 状態名 | 説明 | ボタン表示 |
|---|---|---|
| 停止中 (STOPPED) | カウントアップしておらず、時間表示は固定 | `START` |
| 計測中 (RUNNING) | 1秒ごとにカウントアップ中 | `STOP` |

- ページ読み込み直後の初期状態は「停止中」、時間表示は `00:00:00`。

### 2.1 時間表示フォーマット

- 表示形式: `HH:MM:SS` (例: `00:00:00`)
- 各項目は常に2桁のゼロ埋め表示 (例: 5秒 → `00:00:05`)。
- 表示可能範囲: `00:00:00` 〜 `99:59:59`
- 1秒未満(ミリ秒)の表示は行わない。

### 2.2 レイアウト構成 (drawioに基づく)

drawio上の2つの枠は、同一画面の「停止中(STARTボタン表示)」「計測中(STOPボタン表示)」の
2状態を表すモックアップであり、実際の画面は1つ。

- 画面中央に、時間表示とボタンをまとめた矩形のカード(枠付きコンテナ)を1つ配置する。
- カード内の上部〜中央に、時間表示を大きなフォントサイズで中央揃え表示する(カードの主要な要素として、視認性重視で非常に大きく表示する)。
- カード内の下部中央に、角丸のトグルボタン(START/STOP兼用)を1つ配置する。
- 配色・フォント種別など見た目の詳細指定はなし。実装時に見やすさを優先して適宜決定してよい。

## 3. 動作仕様

### 3.1 ボタン押下時の挙動

- ボタンは1つのみで、押すたびに「開始」「停止」の動作を交互に切り替える(トグル)。

**「停止中」の状態でボタン(`START`)を押した場合:**

1. 時間表示を `00:00:00` にリセットする。
2. カウントアップを開始する(1秒ごとに+1秒)。
3. ボタン表示を `STOP` に変更する。
4. 状態を「計測中」にする。

**「計測中」の状態でボタン(`STOP`)を押した場合:**

1. カウントアップを停止する。
2. その時点の時間表示をそのまま固定表示する(次にSTARTが押されるまで値は変わらない)。
3. ボタン表示を `START` に変更する。
4. 状態を「停止中」にする。

> 備考: STOPで止めた後に再度STARTを押した場合、一時停止からの再開(継続カウント)ではなく、
> 常に `00:00:00` からの新規カウントとなる。

### 3.2 上限到達時 (99:59:59) の挙動

- カウントアップ中に `99:59:59` に到達した場合、自動的に「停止中」状態に遷移する。
  - カウントアップを停止する。
  - 表示は `99:59:59` のまま固定される。
  - ボタン表示を `START` に変更する。
  - この後、再度STARTを押した場合の挙動は 3.1 の「停止中→START」と同じ(`00:00:00` から再スタート)。

### 3.3 リセット機能

- 明示的な「リセットボタン」は実装しない(操作ボタンはSTART/STOP兼用の1つのみ)。
- `00:00:00` に戻す手段は次のいずれか:
  - ページのリロード
  - 「停止中」の状態からSTARTを押す(新規カウント開始時に自動的に0からになる)

## 4. 非機能要件

- 永続化処理(`localStorage`, `sessionStorage`, Cookie等)は一切使用しない。
- 対応デバイス: PC・スマートフォンの両方を想定したレスポンシブ対応を行う(画面幅に応じてレイアウトが破綻しないこと)。
- タイマーの計測方法は、`setInterval` の呼び出し回数のみに依存せず、開始時刻と現在時刻の差分計算(例: `Date.now()` の差分)により実時間との誤差が蓄積しないようにする。

## 5. 技術構成

- 構成: ブラウザで完結する静的Webアプリ(サーバーサイド処理なし)。
- ファイル構成: 役割ごとにファイルを分割する。
  - `index.html` — 画面構造(HTML)
  - `style.css` — 独自スタイル(Tailwindで表現しきれない部分がある場合のみ使用)
  - `script.js` — タイマーのロジック・状態管理・DOM操作
- Tailwind CSS: CDN経由で読み込む(ビルドツール・npmは使用しない)。
  ```html
  <script src="https://cdn.tailwindcss.com"></script>
  ```
- JavaScript: フレームワークを使用しない Vanilla JS で実装する。
